边做边查版 SOP

数学只错一次【新版】任务的操作 SOP（音画同步板书）

面向第一次执行新版任务的人。按后台操作顺序检查和修改字段，每一步都看“入口、操作、通过标准、异常处理、证据回指”。

来源：原 Word《数学只错一次【新版】任务的操作SOP（音画同步板书）》重构；不覆盖原文件。

第一页：任务定义

目标 完成新版音画同步板书任务的数据检查与修改，让讲题模型能按解题过程选择合适板书，并控制哪些内容不要提前出现。
适用范围 任务列表中勾选“新版”的题目；进入编辑页后，内容结构符合新版入参。
完成标准 题干、答案、分析、解答过程、directorGuide 均无明显错误；预览正常；检测提示词复核后能提交或能说明异常原因。
关键理解：新版流程不是精确写一段讲题稿，而是准备一组可被讲题模型调用的板书素材。执行时重点不是“每条板书都会出现”，而是“素材正确、顺序合理、重点可被识别、提前暴露被控制”。
输入后台题目编辑页、清晰题目图片、旧流程提示词输出、Method 检测提示词、directorGuide 检测提示词。
输出提交后的新版任务，或标记为“其他”的异常任务并保留判断证据。
证据链字段内容、编译预览、检测提示词输出、异常截图、联系确认记录。
不做什么不重写教学目标，不扩展题外解法，不为每条板书强行编讲解词，不在无法判断时自行猜测异常原因。

1. 进入任务

1.1 确认进入新版任务
入口
任务列表勾选“新版”，点击【编辑】。
操作
进入编辑页后，先看页面内容结构是否与新版流程一致。
通过标准
能看到新版入参结构，并能继续检查题干、答案、分析、解答过程、directorGuide。
异常处理
如果结构不一致，重新执行进入步骤或刷新；仍失败时按 1.3 判断。
证据回指
编辑页截图；任务是否勾选“新版”；字段区是否出现新版结构。
新人提示
先判断“能不能做”，再开始改字段。
1.2 正常待修改任务
入口
点击【编辑】后进入正常字段编辑页。
操作
按后续流程逐项检查并修改。
通过标准
题目字段可编辑，重点字段可查看或打开编辑框，预览区可用于查看编译效果。
异常处理
如果无法编辑、字段缺失或页面结构明显不对，不继续猜，转 1.3。
证据回指
页面结构截图、字段名、预览区状态。
新人提示
正常任务才进入字段修改；不要在异常页面里强行改。
1.3 异常任务判断
入口
点击【编辑】后页面结构不像正常新版任务。
操作
刷新或重新进入一次，再检查是否同时满足三个条件。
通过标准
同时满足：是“新版”任务；questionMethod 中没有 steps、content 字段；directorGuide 仍为空。
异常处理
三项同时满足：标记异常，标签为“其他”。不是同时满足但仍无法修改：联系鑫宇或刘蒙确认。
证据回指
任务新版状态截图；questionMethod 字段截图；directorGuide 空值截图。
新人提示
异常判断必须看“三项是否同时满足”，不能只凭页面感觉。

2. 检查题干与答案

2.1 题干、答案字段
入口
进入编辑页后，先看题干和答案字段。
操作
检查题干是否完整、答案是否正确。简单错误直接手动修改。
通过标准
题干信息无缺失、无明显 OCR 错误；答案与题目条件和计算过程一致。
异常处理
错误较大时，用清晰题目图片重跑老提示词，得到较正确的题干和答案后再修。复杂题答案暂时无法确定，可等所有字段确认后再回查。
证据回指
题目图片、题干字段、答案字段、老提示词输出。
新人提示
这两个字段沿用老流程，本次新版重点不在这里，但它们错了会影响后面全部字段。

3. 检查三个重点字段

重点字段共三个：questionAnalysis、questionMethod、directorGuide。执行时先看内容是否正确，再看格式是否符合规则。

3.1 分析字段：questionAnalysis
入口
查看 questionAnalysis 字段。
操作
确认解题思路是否正确、是否符合题目条件。简单错误手动改；改动量大时先用老流程提示词产出较正确版本，再人工修。
通过标准
不使用超纲方法；思路正确；内容足够支撑讲题模型理解解题方向；无固定格式要求。
异常处理
如果题目本身信息不清或答案无法判断，先回到题干/答案确认，不直接改 analysis。
证据回指
questionAnalysis 文本、题目图片、必要时的老提示词输出。
新人提示
它提供“思路框架”，不是板书内容；不要把所有板书细节塞进这里。
3.2 解答过程字段：questionMethod
入口
查看 questionMethod.steps，可点击修改按钮打开编辑框，也可在内容框内直接编辑。
操作
逐条检查 content、label、priority、marks、keySegments。修改后用下方编译框预览，保存并刷新后确认编译效果。
通过标准
每条 content 无事实和计算错误；顺序符合讲题逻辑；每条最多一步推导；关键板书标 priority:"key"；关键结论、公式、后续复用结果有标注；label 选择正确。
异常处理
错误太多时，带清晰题目图片走老流程，用老流程输出放入结构化入参，再通过【解题】生成新流程内容；该功能如未上线，按当前团队口径催促或联系确认。
证据回指
questionMethod 原文、修改后 JSON、编译预览、保存刷新后的效果。
新人提示
steps 的顺序就是解题顺序；不是所有 step 都会被课堂使用，它们是给模型选择的板书素材。
字段新人只需要先记住的判断常见错误
content板书上会完整写出来的内容，必须正确、简短、一步一推导。一句里塞多步；计算错；数学元素没有 LaTeX；LaTeX 反斜杠未转义。
label决定写到板书区域哪里。可选：推理、计算、结论、公式、方法、注意、总结。自创标签；含具体数字的步骤没有选推理/计算/结论。
priority核心板书填 key；非核心不填该字段。把所有内容都标 key；或关键结论没有标 key。
marks写出来时就持续高亮的内容。文本必须来自 content。结论没有 answer；公式没有 formula；后续复用的计算结果没有 keyvalue。
keySegments讲题模型可以临时强调的内容，宁多勿少。文本必须来自 content。和 marks 混淆；关键数字和短语未列入。
正文保留示例：正确 step 
{
  "content": "实际比假设多的枚数：$38 - 32 = 6$（枚）",
  "label": "计算",
  "marks": [
    { "text": "6", "style": "keyvalue" }
  ],
  "keySegments": [
    { "index": 1, "text": "38", "desc": "" },
    { "index": 2, "text": "32", "desc": "" },
    { "index": 3, "text": "6", "desc": "" }
  ],
  "priority": "key"
}

这个例子合格，因为它只做一步计算，结果正确，关键中间值用 keyvalue 标出，所有高亮文本都来自 content。

正文保留示例：常见错误 step 
{
  "content": "实际比假设多的枚数：38 - 32 = 6，6枚装盒数：6 \\div 2 = 3",
  "label": "推理",
  "marks": [],
  "keySegments": [],
  "priority": "key"
}

问题：一句里有两步计算；数学表达未按要求放入 LaTeX；这是计算步骤却标成“推理”；关键结果没有 marks 和 keySegments。

3.3 阶段可见性字段：directorGuide
入口
查看 directorGuide 字段。
操作
确认每个阶段 stage 与 visibilityNote 是否能控制“当前阶段不该提前出现什么内容”。一般不大改，除非 questionMethod 做过大幅修改。
通过标准
按题目复杂度设置约 3-6 个阶段；visibilityNote 指向具体对象，例如“最省钱方案结果不该提前出现”。
异常处理
如果写成“计算结果不该出现”这类泛泛表达，需要改成具体内容；如果 method 大改后阶段明显不匹配，需要同步调整。
证据回指
directorGuide JSON、questionMethod 顺序、预览或讲题阶段判断。
新人提示
它不是讲题步骤，而是防止后面答案或结论提前出现在板书上。

4. 用提示词复核

4.1 检测 questionMethod
入口
修改完成后的 questionMethod 或 steps JSON。
操作
打开《Method检测提示词.md》，贴给豆包，选择专家模式，再粘贴待检查字段。
通过标准
检测结果没有硬错误；风险项已人工确认或已修改。
异常处理
检测指出 JSON 结构、LaTeX、计算、label、marks、keySegments 问题时，回到 3.2 修改后重新检查。
证据回指
检测提示词文件名、输入字段、豆包输出结论、最终修改记录。
新人提示
豆包输出不是直接全量覆盖命令，要结合题目事实人工确认。
4.2 检测 directorGuide
入口
修改完成后的 directorGuide JSON。
操作
使用《directorGuide检测提示词.md》复核阶段和提前暴露控制。
通过标准
阶段数量合理；每条 visibilityNote 指向具体内容；与 questionMethod 顺序一致。
异常处理
提示阶段过粗、过细或对象不明确时，回到 3.3 修改后重新检查。
证据回指
检测提示词文件名、输入字段、豆包输出结论、最终修改记录。
新人提示
如果本地没有该提示词文件，先去钉钉文档附件获取，不临时自编规则。

5. 提交或标异常

5.1 正常提交
入口
所有字段修改完成，预览正常，提示词复核完成。
操作
保存，刷新确认编译效果，再提交。
通过标准
字段内容与预览一致；无未处理硬错误；提交前检查表全部通过。
异常处理
保存后预览不一致，先刷新再看；仍不一致，回查 JSON 格式或页面状态。
证据回指
保存后的字段内容、刷新后的预览、检测结果。
新人提示
不要只看编辑框，必须看保存刷新后的编译效果。
5.2 标异常
入口
任务无法按正常流程修改，或同时满足异常任务判断条件。
操作
按异常处理表留证据，必要时联系鑫宇或刘蒙确认。
通过标准
异常原因能回指到字段或页面状态，不是主观感觉。
异常处理
同时满足 1.3 三项条件时，标签选“其他”；其他无法判断情况先确认后处理。
证据回指
页面截图、字段截图、联系确认记录、异常标签。
新人提示
能自己修就修；不能修要说明卡在哪里、证据是什么。

附录：字段规则速查

label 选择

值适用内容
推理用题目信息、假设、关系说明推进思路。
计算含具体数字、算式、结果的步骤。
结论阶段答案或最终答案。
公式需要作为公式展示和持续高亮的内容。
方法方法名称或解法策略。
注意易错提醒。
总结方法归纳或知识总结。

marks 使用规则

style何时使用必须场景
answer最终答案或关键结论。label="结论" 的 step 必须标注答案数值或关键结论。
formula整条公式。label="公式" 的 step 必须标注公式。
keyvalue关键中间计算结果，后续步骤会复用。label="计算" 且结果后续复用时必须标注。

可不加 marks 的情况

  • label="推理" 的中间推导，没有需要持续高亮的对象。
  • label="总结" 的方法归纳。
  • label="注意" 的易错提醒。
  • 计算结果不会被后续步骤再用到的中间步骤。
